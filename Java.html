Introduction to Java Programming

Introduction to Java Programming: Java is a high-level object-oriented programming language developed in the early 90s by James Gosling. The java programming language was called OAK. OAK was originally developed for interactive televisions; however, the project was later redesigned to become what it is today. Nowadays Java is known as the most popular programming language and is used for things such as internet programming, mobile devices, games, and e-business solutions.

Why use Java?

Java is one of the most popular programming languages and platforms in the world and is often the first programming language taught to new computer science students. This is because java was built on one core idea, the ability to “Write Once, Run Anywhere”. This means that java can be written on one system and run on all available operating systems that support java, giving the developer the ability to work on multiple platforms.

    Java Terminology

    List of Java Terms

  • Class: A blueprint or prototype from which objects are created. An example might be a “Customer” class. The class can define attributes such as “name”, “customer number”, “address”. The class also defines behaviors (methods) for objects created with the class. For instance, methods might be used to set the customer’s address, change the customer’s address, and get the address.

  • Objects:An entity that has state and behavior also known as an instance. For example a bike that has states (brand, # of gears, bicycle type) and behaviors (pedal, break, turn). It can be physical (bike, person) or logical (date, checking account).

  • Method: Functions that belong to a class. They are used to operate on the internal state of an object and the object-to-object communications.

  • Instance variable:A variable which is declared in a class but outside of constructors, methods, or blocks.

  • Identifiers: A programmer-defined name for a variable, method, class, or interface.

  • Arrays: A fixed-size object that can hold zero or more items of the array’s declared type.

  • Access Modifiers:Controls the access level. Some access modifiers include public, private, and protected.

  • Public Is a Java keyword which declares a member's access as public. Public members are visible to all other classes.

  • Private:Is an access modifier used for attributes, methods, and constructors, making them only accessible within the declared class.

  • Protected: Is an access modifier used for attributes, methods and constructors, making them accessible in the same package and subclasses.

  • Non-Access Modifier:Do not control access level but provides other functionality. Some non-access modifiers include static, final, abstract, synchronized, volatile, transient, and native.

  • Compile: The action by a program (the compiler) of converting source code to a binary file that contains the low-level instructions that the computer’s processor will execute.

  • Iterate: Iteration is when the same procedure is repeated multiple times

  • Void: A keyword that indicates a method does not return a value:

    • Java Data Types

    Specifies the size and type of values that can be stored in an identifier. The Java language is rich in its data types. Different data types allow you to select the type appropriate to the needs of the application. Data types can be split up into two main categories primitive and non-primitive data types.

    Data types of classification:

    • Primitive data types
      • Are predefined in the java.
      • Always have a value.
      • Start with a lower-case letter.
      • The size of the primitive types is determined by data types itself.
      • There are eight different primitive types in java: Boolean, byte, char, double, float, int, long and short.

      Data Type Size Description
      byte 1 byte Stores whole numbers from -128 to 127
      short 2 bytes Stores whole numbers from -32,768 to 32,767
      int 4 bytes Stores whole numbers from -2,147,483,648 to 2,147,483,647
      long 8 bytes Stores whole numbers from -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
      float 4 bytes Stores fractional numbers. Sufficient for storing 6 to 7 decimal digits
      double 8 bytes Stores fractional numbers. Sufficient for storing 15 decimal digits
      boolean 1 bit Stores true or false values
      char 2 bytes Stores a single character/letter or ASCII values

    • Non-primitive data types
      • Are not defined by java.
      • can be a null value.
      • Start with an upper-case letter.
      • Non-primitive data types all have the same size.
      • Some of the non-primitive data types include: Strings, arrays, classes, and interface.

    Sting type: This data type is used to store consecutive characters. Strings are treated as objects.

    Arrays This is a data type used to store data in a consecutive manner. It can store any type of data as the size of the array is also declared by the programmer.

    Classes: A class is a user-defined blueprint or prototype from which objects are created. It represents the set of properties or methods that are common to all objects of one type.

    Interface: An interface is a completely "abstract class" that is used to group related methods with empty bodies.

    Basic Syntax

    The syntax of Java refers to the set of rules defining how a Java program is written and interpreted. It is important that syntax is correct as to prevent any errors in the code.

    List of syntax rules

  • Case sensitivity: The java programming language is case sensitive. This means that identifiers with different cases such as Apple and apple would have different meanings.

  • Methods: Methods should be verbs, in mixed case with the first letter lowercase, with the first letter of each internal word capitalized.

  • Variables: The variable names should not start with a dollar (‘$’) or underscore(‘_’). One character variable name must be avoided and should be mnemonic.

  • Comments: Comments can be used to explain Java code, and to make it more readable. It can also be used to prevent execution when testing an alternative code.

  • Constant variables: The name of a constant variable must have all the letters capital and if it has more words at name should be separated by an underscore (‘_’).

  • Semicolons: The semicolon (‘;’) is a character that we see almost at every line of code. It shows the compiler where an instruction ends and where the next instruction begins. Some statements have a special characteristic which is that they are self-close.

  • Brackets, Braces, parentheses:
      • Brackets are used to index into an array.

      • Braces are used for three purposes: one to group statements and declarations. The contents of a class or interface are enclosed in braces. The second is for enclosing methods, and constructor bodies are enclosed in braces. The third is to group the statements in an if statement, a loop, or other control structures.

      • Parentheses are used for two purposes: one is to control the order of operations in an expression and second use is to supply parameters to a constructor or method.

    Control Statements

    A control statement is a statement in java that determines whether the other statements will be executed or not. It can also be referred to as a decision-making statements, looping statements, or branching statement. The purpose of the control statement is to control the flow of the program.

    List of control statements:

  • If statement: The if statement is used to specify whether a block of code will be executed, if the statement is true the code block will execute.

  • If-else statement: The if-else statement is used to specify whether a block of code will be executed, if the statement is true the code will execute otherwise the else block will execute.

  • Nested if statement: The nested if-else statement is similar to an if statement. However, a nested if statement executes one condition from multiple statements.

  • Switch statement: The switch statement is used to execute a block of code depending on different cases.

  • For loop: The for loop statement is used to execute a sequence of statements multiple times. A for loop is useful when you know how many times a task is to be repeated.

  • While loop: The While loop statement is used to repeat a statement or group of statements while a given condition is true. It tests the condition before executing the loop body.

  • Do-while loop: The do-while loop statement is similar to a while loop statement, except it tests the condition at the end of the loop body.

  • Break statement: The break statement is used to end the loop or switch statement.

  • Continue statement: The continue statement is used to break one iteration (in the loop), if a specified condition occurs, and continues with the next iteration in the loop.

    • Object-Oriented Programming

    Object-Oriented programming OOP is a programming paradigm that revolves around the concept of classes and objects. OOP is based around 4 core principles: polymorphism, inheritance, encapsulation, and abstraction.

    Four principles of OOP:

  • Polymorphism: Polymorphisms means having many forms. In java subclasses of a class can define their own unique behaviors and yet share some of the same functionality of the parent class. For example, a class called Animal that has a method called animalSound(). Subclasses of Animals could be Cats, Dogs. They also have their own implementation of an animal sound (the dog barks, and the cat meows).

  • Inheritance: Inheritance is when one class is able to inherit the features (fields and methods) of another class. There are two keywords to know when it comes to inheritance subclass (child) and superclass (parent). One example of inheritance is a class called Car (subclass) inheriting the features of a class called vehicle (superclass).

      • Subclass (child): The class that inherits features from another class is called a subclass.

      • Superclass (parent): The class that’s features are being inherited from.
  • Encapsulation: Encapsulation is when the variables of a class will be hidden from other classes and can be accessed only through the methods of their current class.

  • Abstraction: Abstraction is the process of hiding certain details and showing only essential information to the user. Abstraction can be achieved with either abstract classes or interfaces
