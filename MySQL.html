Introduction to MySQL

MySQL is often referred to as the most used Open-Source Relational SQL database management system or (RDBMS). MySQL was developed in the mid-90s by Michael Widenius, David Axmark and Allan Larsson. The name MySQL comes from a combination of "My", the name the primary developer Michael Widenius's daughter, and "SQL", which stands for Structured Query Language. MySQL was written in C and C++ and can work with a variety of platforms such as Windows, MacOS, Unix, and Linux to name a few. In order to fully understand MySQL, the user needs to have an understanding of databases, RDBMS, and SQL.

This short video goes into more detail of how and why it was developed: History of MySQL

How to install MySQL?

Installing MySQL on a Mac computer

  1. Open MySQL and scroll to the bottom of the page.

  2. Click on MySQL Community (GPL) Downloads » MySQL Community Server.

  3. After clicking MySQL Community server a new window will pop up, select the MacOS system. Below select DMG Archive. This will take you to a new page scroll to the bottom and select No thanks, just start my download to begin downloading.

  4. Open the download file and follow the instructions to complete the installation process. When it asks to create a password create a new password. This will be used again later.

  5. Next step is to download the graphical user interface that will allow us to manage the database system. Repeat Steps 1 and 2.

  6. Click on MySQL workbench and a new window will pop up select the MacOS system. Below select DMG Archive. This will take you to a new page scroll to the bottom and select No thanks, just start my download to begin downloading.

  7. Open the MySQL workbench and click on the plus button next to MySQL connections to create a new connection.

  8. A new window will pop up, select the connection name box and create a new name.

  9. Select the store in key Change and enter the password used during the installation process. Leave all other options as default.

  10. Finally in order to open the workbench go back to the start page and select the new connection that has been created.

Installing MySQL on a Windows PC

  1. Open MySQL and scroll to the bottom of the page.

  2. Click on MySQL Community (GPL) Downloads » MySQL Community Server.

  3. After clicking MySQL Community server a new window will pop up, select the Windows operating system. Below on the box that says MySQL installer on windows. This will take you to a new page scroll and select the first installer on the list. Next scroll to the bottom and select No thanks, just start my download to begin downloading.

  4. Open the download file and follow the instructions to complete the installation process.

  5. On the part titled Accounts and Roles it will ask to create a password, create a new password. This will be used again later.

  6. Continue the installation process until a section titled Connect to server appears. Type in the password created earlier to proceed to the next step.

  7. After finally completing the installation process a command line window will appear. Exit this and open MySQL workbench.

  8. Open the MySQL workbench and click on the plus button next to MySQL connections to create a new connection. A new window will pop up, select the connection name box and create a new name.

  9. Select the store in key Change and enter the password used during the installation process. Leave all other options as default.

  10. Finally in order to open the workbench go back to the start page and select the new connection that has been created.

What is a Database?

A database is an organized collection of data that is stored on a computer system. Every database has at least one or more APIs (Application Programming Interface) that allow the user to create, access, manage, search, and replicate the data being stored in it. Databases are used all over the world in a variety of different places like in hospitals, banks, and schools.

Relational DataBase Management System (RDBMS)

Relational DataBase Management System (RDBMS): These days we use something called relational database management systems RDBMS to store and manage large quantities of data. RDBMS is given its name because of the way the data is stored in the software. Each piece of data is grouped into separate tables that have relations which are created using something called primary and foreign keys.

Relational DataBase Management System Vocabulary:

  • Database: A Database is an organized collection of data stored grouped into tables and stored on a computer system.

  • Tables: A table is a collection of related data held in a table format within a database. Each table consists of rows and columns similar to the design of a spreadsheet.

  • Row: A row is a collection related data, for instance all the data for one specific customer. A row is sometimes to a tuple or record.

  • Columns: A column is a collection of data of the same type, for instance a phone number.

  • Primary Key: A primary key is used to uniquely identify an entry in a table.

  • Foreign Key: A foreign key is used to establish a link or relationship between two different tables.

  • Composite Key: A composite key is a key that can be selected as a primary key of the table.

  • Alternate key: Alternate keys are the remaining candidate keys, not currently selected as a primary key of the table.

  • Composite key: A composite key is a group of two or more columns that identifies each row of a table uniquely.

  • Unique Key: A unique key is similar to a primary key and does not allow duplicates in the column

  • Super Key: A super key is a set of columns on which all columns of the table are functionally dependent. It is a set of columns that uniquely identify each row in a table.

  • Referential Integrity: Referential Integrity makes sure that a foreign key value always points to an existing row.

  • Index: An index is a quick way of locating data that is stored within the database.

    MySQL vs SQL

    MySQL

  • MySQL is RDBMS that is used to store, retrieve, modify, and manage a database using SQL.

  • MySQL is database software. It uses SQL to query the database.

  • It is readily available through download and installation find more details MySQL download.

  • MySQL helps to manage relational databases using SQL.

    • SQL

  • SQL or structure Structured Query Language. It is used for storing, manipulating, and retrieving data in databases.

  • SQL is a query language. SQL code and commands are used in various DBMS and RDBMS systems including MYSQL.

  • In order to use SQL, the user needs to learn the SQL language. SQL Tutorial:

  • SQL helps to manage the data in the relational database.

    • MySQL/SQL Commands

    In order to store, retrieve, modify, and manage a database the user must learn SQL commands. These commands can be split up into four categories: Data Definition Language (DDL), Data Query Language (DQL), Data Manipulation Language (DML), and Data Control Language (DCL).

    Examples of DDL commands:

    Data Definition Language: DDL refers to the set of SQL commands that can create and manipulate the structure of the database.

  • Create table: The create table statement is used to create a table in a database organized into rows and columns.

  • Alter table: The alter table statement is used to add, remove, delete, or modify columns in an existing table.

  • Drop table: The drop table statement is used to remove a table definition and all the data, indexes, triggers, constraints, and permission specifications for that table.

  • Truncate table: Truncate table delete the data inside a table, but not the table itself.

    • Examples of DML commands:

    Data Manipulation Language: DML deals with data manipulation and includes the most common SQL statements.

  • Insert:The insert into statement is used to add a new record to a table.

  • Delete:The delete from statement is used to delete rows from a table.

  • Update:The update where statement is used to update or modify data in an existing table.

    • Examples of DQL commands:

    Data Query Language: DQL is used as a way to perform a query on the data within schema objects. A schema object is a logical structure created by the users.

  • Select:The select statement is used to retrieve information from the database.

    • Examples of DCL commands:

    Data Control Language: DCL is used to control access to data stored in a database.

  • Grant:The grant command gives user’s access privileges to the database.

  • RevokeThe revoke command withdraws the user’s access privileges given by using the grant command.
